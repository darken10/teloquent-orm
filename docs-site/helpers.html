Helpers CRUD

Des raccourcis Eloquent fréquents : trouver-ou-créer, mettre-à-jour-ou-créer, upsert en masse et incrémentation.

firstOrNew

Cherche le 1er enregistrement correspondant aux attributs ; sinon retourne une instance non sauvegardée.

const user = await User.firstOrNew(
  { email: "z@a.com" },   // critères de recherche
  { name: "Inoussa ZERBO" }     // valeurs additionnelles si création
);
user.$exists; // false si nouveau
await user.save();

firstOrCreate

Comme firstOrNew, mais persiste immédiatement si absent.

const user = await User.firstOrCreate(
  { email: "z@a.com" },
  { name: "Inoussa ZERBO" }
);
// Trouvé -> renvoyé tel quel ; absent -> créé. Pas de doublon.

updateOrCreate

Met à jour l'enregistrement correspondant, ou le crée.

const setting = await Setting.updateOrCreate(
  { key: "theme" },        // recherche
  { value: "dark" }        // valeurs à écrire
);

upsert (en masse)

Insère plusieurs lignes, ou les met à jour en cas de conflit sur les colonnes uniques. Génère le bon SQL par dialecte (ON CONFLICT pour SQLite/PostgreSQL, ON DUPLICATE KEY UPDATE pour MySQL).

await User.upsert(
  [
    { email: "a@b.com", name: "A", votes: 1 },
    { email: "c@d.com", name: "C", votes: 5 },
  ],
  ["email"],          // colonne(s) de conflit (clé unique)
  ["name", "votes"]   // colonnes à mettre à jour si conflit
);
La colonne de conflit doit posséder une contrainte unique (ou être clé primaire) dans le schéma.

increment / decrement

Modifie une colonne numérique directement en base (et localement sur l'instance).

// Sur une instance
await user.increment("votes");       // +1
await user.increment("votes", 5);    // +5
await user.decrement("credits", 2);  // -2
user.votes;                          // mis à jour localement

// En masse
await User.query().where("active", true).increment("votes", 10);
En masse sur un modèle avec timestamps, increment/decrement mettent aussi à jour updated_at.
