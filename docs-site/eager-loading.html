Eager loading & withCount

Chargez les relations en une requête par relation pour éviter le problème N+1, y compris sur plusieurs niveaux, et comptez des relations sans les charger.

Le problème N+1

// ❌ N+1 : 1 requête pour les users + 1 requête par user
const users = await User.all();
for (const u of users) {
  const posts = await u.posts().getResults(); // une requête à chaque tour
}

with() : eager loading

// ✅ 2 requêtes au total : 1 pour users, 1 pour tous leurs posts
const users = await User.with("posts").get();
for (const u of users) {
  const posts = u.getRelation("posts"); // déjà chargé, aucune requête
}

// Plusieurs relations
await User.with("posts", "profile", "roles").get();

Eager loading imbriqué

Utilisez la notation pointée pour charger en cascade, sur n'importe quel type de relation et n'importe quelle profondeur.

const users = await User.with("posts.comments", "posts.tags").get();

for (const user of users) {
  for (const post of user.getRelation("posts")) {
    post.getRelation("comments"); // chargé
    post.getRelation("tags");     // chargé (relation N-N)
  }
}
Chaque niveau est résolu en une seule requête (un whereIn sur l'ensemble des clés), même imbriqué.

withCount()

Ajoute un attribut <relation>_count à chaque modèle, sans charger les enregistrements liés.

const posts = await Post.query().withCount("comments", "tags").get();

posts.first().getAttribute("comments_count"); // 12
posts.first().getAttribute("tags_count");     // 3

// Apparaît aussi dans la sérialisation
posts.first().toJSON().comments_count;        // 12

withCount fonctionne pour hasMany, hasOne, belongsTo et belongsToMany, et respecte les soft deletes du modèle lié.

Combiner avec le query builder

const users = await User.query()
  .where("is_active", true)
  .with("posts.comments")
  .withCount("posts")
  .orderBy("name")
  .paginate(1, 20);

Charger après coup (lazy eager)

const user = await User.find(1);
await user.load("posts");
user.getRelation("posts");

Performance

  • with : 1 requête supplémentaire par relation (et par niveau), au lieu d'une par parent.
  • withCount : 1 requête agrégée groupée par relation.
  • Les clés sont comparées de façon tolérante au type (certains drivers renvoient les bigint en chaînes).
