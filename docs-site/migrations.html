Migrations

Le Migrator applique et annule des changements de schéma de façon versionnée, via une table migrations (name, batch).

Définir des migrations

Une migration est un objet avec un name unique et des fonctions up/down.

import { Schema, type MigrationEntry } from "teloquent-orm";

export const migrations: MigrationEntry[] = [
  {
    name: "001_create_users",
    up:   () => Schema().create("users", (t) => {
      t.increments("id");
      t.string("name");
      t.timestamps();
    }),
    down: () => Schema().dropIfExists("users"),
  },
  {
    name: "002_add_email_to_users",
    up:   () => Schema().table("users", (t) => { t.string("email").nullable(); }),
    down: () => Schema().table("users", (t) => { t.dropColumn("email"); }),
  },
];
Préfixez les noms par un numéro/horodatage (001_, 20260621_) : l'ordre du tableau définit l'ordre d'exécution.

Exécuter

import { Migrator } from "teloquent-orm";
import { migrations } from "./migrations.js";

const m = new Migrator();

await m.run(migrations);      // applique les migrations en attente (nouveau batch)
await m.rollback(migrations); // annule le dernier batch
await m.reset(migrations);    // annule toutes les migrations
await m.refresh(migrations);  // reset + run (recrée tout)
await m.status(migrations);   // [{ name, ran, batch }]

Comportement

  • Idempotent : run n'applique que les migrations non encore exécutées.
  • Batches : chaque run crée un nouveau numéro de batch.
  • Rollback ciblé : rollback n'annule que le dernier batch (dans l'ordre inverse).
  • La table migrations est créée automatiquement si absente.

Statut

const status = await m.status(migrations);
// [
//   { name: "001_create_users",        ran: true,  batch: 1 },
//   { name: "002_add_email_to_users",  ran: false, batch: null },
// ]

Script CLI maison

Teloquent fournit la mécanique ; vous l'exposez comme vous le souhaitez (ex. via npm scripts).

migrate.ts 
import { ConnectionManager, Migrator } from "teloquent-orm";
import { migrations } from "./migrations.js";

await ConnectionManager.addConnection({ driver: "sqlite", database: "app.sqlite" });
const m = new Migrator();

const cmd = process.argv[2] ?? "run";
if (cmd === "run")      console.log("appliquées :", await m.run(migrations));
if (cmd === "rollback") console.log("annulées :", await m.rollback(migrations));
if (cmd === "refresh")  await m.refresh(migrations);
if (cmd === "status")   console.table(await m.status(migrations));

await ConnectionManager.closeAll();
tsx migrate.ts run
tsx migrate.ts rollback
tsx migrate.ts status

Classe Migration (alternative)

import { Migration } from "teloquent-orm";

class CreateUsers extends Migration {
  async up()   { await this.schema.create("users", (t) => { t.increments("id"); }); }
  async down() { await this.schema.dropIfExists("users"); }
}
