Événements

Réagissez au cycle de vie des modèles (création, mise à jour, suppression) via des observers, façon « model events » d'Eloquent.

S'abonner

import { ModelEvents } from "teloquent-orm";

ModelEvents.on(User, "creating", (user) => {
  user.email = String(user.email).toLowerCase();
});

ModelEvents.on(User, "created", (user) => {
  console.log("Nouvel utilisateur :", user.getKey());
});

ModelEvents.on(User, "deleted", async (user) => {
  await AuditLog.create({ action: "user.deleted", target: user.getKey() });
});

Événements disponibles

ÉvénementDéclenché…
creatingavant l'INSERT
createdaprès l'INSERT
updatingavant l'UPDATE
updatedaprès l'UPDATE
deletingavant la suppression (ou soft delete)
deletedaprès la suppression

Callbacks asynchrones

Les callbacks peuvent être async : ils sont attendus avant que l'opération se poursuive.

ModelEvents.on(Order, "creating", async (order) => {
  order.reference = await generateUniqueRef();
});

Cas d'usage fréquents

  • Normalisation : forcer un email/slug en minuscules à la création.
  • Valeurs dérivées : générer une référence, un UUID, un slug.
  • Audit / journalisation : tracer les suppressions et modifications.
  • Invalidation de cache après updated/deleted.
Les événements sont liés à la classe : un observer sur une classe ne se déclenche que pour les instances de cette classe.
