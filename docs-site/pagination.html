Pagination

Paginez les résultats avec total et nombre de pages, ou de façon « simple » sans COUNT, et traitez de gros volumes par lots.

paginate()

const page = await Post.query().orderBy("created_at", "desc").paginate(1, 15);

page.data;        // Collection<Post> (15 éléments)
page.total;       // nombre total d'enregistrements
page.perPage;     // 15
page.currentPage; // 1
page.lastPage;    // nombre de pages
page.from;        // index du 1er élément (1)
page.to;          // index du dernier élément (15)

Structure renvoyée

ChampTypeDescription
dataCollectionLes enregistrements de la page
totalnumberTotal (via COUNT)
perPagenumberTaille de page
currentPagenumberPage courante
lastPagenumberDernière page
from / tonumberBornes des indices

simplePaginate()

Plus léger : ne fait pas de COUNT, indique seulement s'il existe une page suivante.

const page = await Post.query().simplePaginate(1, 15);
page.data;     // Collection
page.hasMore;  // boolean
page.perPage;
page.currentPage;

forPage()

// Équivaut à .offset((page-1)*perPage).limit(perPage)
await Post.query().forPage(2, 10).get();

chunk() : traitement par lots

Pour parcourir un grand nombre de lignes sans tout charger en mémoire.

await Post.query().where("archived", false).chunk(500, async (rows, page) => {
  for (const post of rows) {
    await reindex(post);
  }
  console.log(`lot ${page} traité (${rows.length} lignes)`);
});

Exemple : endpoint d'API

app.get("/posts", async (req, res) => {
  const page = Number(req.query.page ?? 1);
  const result = await Post.query()
    .where("published", true)
    .with("author")
    .orderBy("created_at", "desc")
    .paginate(page, 20);

  res.json({
    data: result.data.toJSON(),
    meta: { total: result.total, page: result.currentPage, lastPage: result.lastPage },
  });
});
La pagination respecte les scopes et les soft deletes : le total exclut bien les enregistrements supprimés.
