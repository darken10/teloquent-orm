Soft deletes

Les soft deletes marquent un enregistrement comme supprimé (colonne deleted_at) au lieu de l'effacer réellement. Les enregistrements supprimés sont alors exclus des requêtes par défaut.

Activer

class Post extends Model {
  static softDeletes = true;
  static deletedAtColumn = "deleted_at"; // par défaut
}

Ajoutez la colonne dans le schéma :

await Schema().create("posts", (t) => {
  t.increments("id");
  t.string("title");
  t.timestamps();
  t.softDeletes(); // ajoute deleted_at (nullable)
});

Supprimer & restaurer

const post = await Post.findOrFail(1);

await post.delete();        // positionne deleted_at (pas de DELETE réel)
post.trashed();             // true

await post.restore();       // remet deleted_at à null
post.trashed();             // false

await post.forceDelete();   // suppression définitive (DELETE réel)

Interroger

await Post.query().get();         // exclut les supprimés (par défaut)
await Post.find(1);               // ignore les supprimés

await Post.withTrashed().get();   // inclut les supprimés
await Post.onlyTrashed().get();   // seulement les supprimés

await Post.withTrashed().where("id", 1).first(); // retrouver un supprimé
Le filtre deleted_at is null est appliqué automatiquement via un scope interne. withTrashed() le retire, onlyTrashed() l'inverse.

Suppression en masse

// Soft delete en masse (UPDATE deleted_at) si le modèle l'active
await Post.query().where("spam", true).delete();

// Suppression réelle en masse
await Post.query().where("spam", true).forceDelete();

Comptage

await Post.query().count();           // hors supprimés
await Post.withTrashed().count();     // tous
await Post.onlyTrashed().count();     // supprimés uniquement

Interaction avec les relations

Les compteurs withCount et les sous-requêtes whereHas respectent les soft deletes du modèle lié : un enfant supprimé n'est pas compté ni considéré comme « existant ».
