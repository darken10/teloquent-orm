Déployer cette documentation

Ce site est entièrement statique (HTML/CSS/JS), sans étape de build. Déployez le dossier docs-site/ sur n'importe quel hébergement statique.

Structure

docs-site/
├── index.html            # accueil
├── installation.html
├── connexions.html
├── ... (une page par rubrique)
└── assets/
    ├── styles.css        # thème clair/sombre
    └── app.js            # header, sidebar, recherche, TOC, copy…

Aperçu local

npx serve docs-site
# ou
python3 -m http.server -d docs-site 8080

GitHub Pages

# Publier le dossier docs-site sur la branche gh-pages
npx gh-pages -d docs-site
# puis : Settings → Pages → Source = branche gh-pages

Alternative via Actions :

.github/workflows/docs.yml 
name: Deploy docs
on: { push: { branches: [main] } }
permissions: { pages: write, id-token: write }
jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: { name: github-pages }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/upload-pages-artifact@v3
        with: { path: docs-site }
      - uses: actions/deploy-pages@v4

Netlify

netlify deploy --dir=docs-site --prod
# ou via l'UI : "Publish directory" = docs-site, aucun build command

Vercel

# Framework Preset: Other — Output Directory: docs-site — Build Command: (vide)
vercel --prod

Fonctionnement 100 % hors-ligne

La coloration syntaxique charge highlight.js via CDN. Pour un site sans dépendance externe, téléchargez highlight.min.js, le langage TypeScript et un thème CSS dans assets/, puis remplacez les URL CDN dans assets/app.js par des chemins locaux. Tout le reste (navigation, recherche, thème, TOC) fonctionne déjà sans réseau.

Le site n'a besoin d'aucun serveur ni base de données : c'est une documentation statique. Hébergez-la gratuitement sur GitHub Pages, Netlify, Vercel, Cloudflare Pages…
