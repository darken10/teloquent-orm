Définir un modèle

Un modèle Teloquent est une classe qui étend Model. Comme Eloquent, il est « schemaless » : on déclare les propriétés pour le typage, les attributs sont stockés dynamiquement.

Modèle minimal

import { Model } from "teloquent";

class User extends Model {
  declare id: number;
  declare name: string;
  declare email: string;
}
Le mot-clé declare indique à TypeScript le type d'un attribut sans générer de champ réel. C'est ce qui permet au Proxy interne d'intercepter user.name et user.name = ….

Configuration

Toutes ces propriétés statiques sont optionnelles.

class User extends Model {
  static table = "users";          // déduit sinon : "User" -> "users"
  static primaryKey = "id";        // clé primaire
  static incrementing = true;      // clé auto-incrémentée
  static timestamps = true;        // created_at / updated_at automatiques
  static connectionName = "default";
  static fillable = ["name", "email"]; // assignables en masse
  static guarded = ["id"];             // ou : protégés (si fillable vide)
  static casts = { is_active: "boolean", meta: "json" };
}

Casts (conversion de types)

Les casts convertissent automatiquement les attributs à la lecture et sérialisent à l'écriture.

CastLecture renvoie
int, floatnumber
booleanboolean (0/1 → false/true)
stringstring
jsonobjet/array (parse) ; stocké en chaîne JSON
date, datetimeDate
class User extends Model {
  static casts = { is_active: "boolean", meta: "json" };
  declare is_active: boolean;
  declare meta: { theme: string } | null;
}

const u = await User.create({ is_active: true, meta: { theme: "dark" } });
u.is_active;   // true (boolean)
u.meta.theme;  // "dark" (objet)

Accessors & Mutators

Un accessor calcule une valeur dérivée ; un mutator transforme une valeur avant stockage. Conventions : getXxxAttribute() et setXxxAttribute().

class User extends Model {
  declare first_name: string;
  declare last_name: string;

  // Accessor : user.full_name
  getFullNameAttribute(): string {
    return `${this.getAttribute("first_name")} ${this.getAttribute("last_name")}`;
  }

  // Mutator : normalise l'email en minuscules
  setEmailAttribute(value: string) {
    return value.toLowerCase();
  }
}

const u = new User({ first_name: "Inoussa ZERBO", last_name: "Traoré" });
u.full_name; // "Inoussa ZERBO Traoré"
Les accessors « purs » (sans colonne en base) doivent être ajoutés à static appends pour apparaître dans toJSON(). Voir Sérialisation.

Mass assignment

Pour se protéger des assignations massives non désirées, seules les colonnes fillable sont remplies par create()/fill(). Si fillable est vide, c'est guarded qui s'applique (liste noire).

class User extends Model {
  static fillable = ["name", "email"];
}

User.create({ name: "A", email: "a@b.com", is_admin: true });
// is_admin est ignoré (non fillable)

// Forcer (contourne fillable)
const u = new User();
u.forceFill({ is_admin: true });

Décorateurs (optionnels)

Sucre syntaxique équivalent aux propriétés statiques.

import { Model, table, casts } from "teloquent";

@table("users")
@casts({ is_active: "boolean" })
class User extends Model {
  declare name: string;
}

Dirty tracking

Teloquent suit les attributs modifiés depuis le dernier chargement/sauvegarde : save() n'émet un UPDATE que sur les colonnes réellement changées.

const u = await User.find(1);
u.isDirty();        // false
u.name = "Nouveau";
u.isDirty();        // true
u.getDirty();       // { name: "Nouveau" }
await u.save();     // UPDATE users SET name = ? WHERE id = ?
u.isDirty();        // false
