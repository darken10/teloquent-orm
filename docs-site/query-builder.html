Query Builder

Une API fluide pour construire des requêtes SQL. Accessible via Model.query() ou directement Model.where(...). Toutes les valeurs sont liées en paramètres (anti-injection).

Sélection & conditions

User.query()
  .select("id", "name", "email")
  .where("age", ">", 18)
  .orWhere("role", "admin")
  .orderBy("name", "desc")
  .limit(10)
  .offset(20);

Variantes de where

MéthodeSQL généré
where("a", 1)a = ?
where("a", ">", 1)a > ?
orWhere(...)or ...
whereIn("a", [1,2])a in (?, ?)
whereNotIn("a", [...])a not in (...)
whereNull("a") / whereNotNull("a")a is [not] null
whereBetween("a", [1, 9])a between ? and ?
whereColumn("a", ">", "b")a > b (sans valeur)
whereRaw("len(x) > ?", [3])expression brute

Groupes imbriqués (closures)

Passez une closure pour produire un groupe parenthésé.

await User.query()
  .where("active", true)
  .where((q) => q.where("role", "admin").orWhere("role", "editor"))
  .get();
// ... where active = ? and (role = ? or role = ?)

Tri, regroupement, limites

User.query()
  .orderBy("created_at", "desc")
  .latest()                 // alias de orderBy("created_at", "desc")
  .groupBy("status")
  .having("total", ">", 100)
  .limit(15).offset(30);    // ou .take(15).skip(30)

Jointures

await User.query()
  .join("posts", "posts.user_id", "=", "users.id")
  .leftJoin("profiles", "profiles.user_id", "=", "users.id")
  .select("users.*")
  .get();

Agrégats

await User.query().count();          // number
await User.query().where("active", true).count();
await User.query().max("age");
await User.query().min("age");
await User.query().sum("votes");
await User.query().avg("score");
await User.query().exists();         // boolean

Récupération

await User.query().get();            // Collection<User>
await User.query().first();          // User | null
await User.query().firstOrFail();    // User
await User.query().value("email");   // valeur d'une colonne du 1er résultat
await User.query().pluck("name");    // string[]

Écriture

await User.query().insert({ name: "A" });
await User.query().insert([{ name: "A" }, { name: "B" }]);  // en masse
const id = await User.query().insertGetId({ name: "A" });
await User.query().where("id", 1).update({ name: "X" });
await User.query().where("spam", true).delete();
await User.query().where("id", 1).increment("votes", 2);
await User.query().where("id", 1).decrement("stock");

selectRaw

await User.query()
  .select("status")
  .selectRaw("count(*) as total")
  .groupBy("status")
  .get();

Filtrer par relation (whereHas)

await User.query().has("posts").get();
await User.query().doesntHave("posts").get();
await User.query().whereHas("posts", (q) => q.where("published", true)).get();

Débogage : voir le SQL

const { sql, bindings } = User.query().where("age", ">", 18).orderBy("name").toSql();
// SQLite/MySQL : ... where "age" > ? order by "name" asc   bindings: [18]
// PostgreSQL  : ... where "age" > $1 ...
Le query builder produit le bon dialecte automatiquement : quoting des identifiants et placeholders (? vs $1) selon la connexion.
