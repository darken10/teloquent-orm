Scopes

Les scopes encapsulent des contraintes de requête réutilisables. Teloquent gère les scopes locaux (invoqués à la demande) et globaux (appliqués automatiquement).

Scopes locaux

Définissez une méthode statique scopeXxx(query, ...args), puis invoquez-la via scope("xxx").

class Post extends Model {
  static scopePublished(q) {
    q.where("published", true);
  }
  static scopeOfType(q, type: string) {
    q.where("type", type);
  }
}

await Post.scope("published").get();
await Post.scope("ofType", "article").get();

// Combinable avec le query builder
await Post.query().scope("published").where("title", "like", "A%").orderBy("created_at").get();

Scopes globaux

Un scope global s'applique à toutes les requêtes du modèle. Idéal pour le multi-tenant ou un filtre métier permanent.

class Tenant extends Model {
  declare active: boolean;
}

// À enregistrer une fois (au démarrage)
Tenant.addGlobalScope("active", (q) => q.where("active", true));

await Tenant.query().get();   // filtre "active = true" appliqué automatiquement
await Tenant.all();           // idem
Tenant.query().toSql().sql;   // ... where "active" = ?
Les scopes globaux sont hérités le long de la chaîne de classes : un scope sur une classe de base s'applique aussi à ses sous-classes.

Exemple complet : multi-tenant

let currentTenantId = 1;

class Invoice extends Model {
  declare tenant_id: number;
}
Invoice.addGlobalScope("tenant", (q) => q.where("tenant_id", currentTenantId));

// Toutes les requêtes sont automatiquement filtrées sur le tenant courant
await Invoice.query().get();
await Invoice.where("status", "paid").get();

Soft deletes = un scope global interne

Le filtre deleted_at is null des soft deletes est implémenté comme un scope appliqué automatiquement, que withTrashed()/onlyTrashed() savent désactiver.
