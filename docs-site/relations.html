Relations

Comme Eloquent, une relation est une méthode du modèle qui retourne un objet relation. Teloquent gère hasOne, hasMany, belongsTo et belongsToMany.

Vue d'ensemble

RelationTypeExemple
hasOne1-1un User a un Profile
hasMany1-Nun User a plusieurs Post
belongsToN-1 (inverse)un Post appartient à un User
belongsToManyN-Nun User a plusieurs Role (table pivot)

Définition

class User extends Model {
  posts()   { return this.hasMany(Post); }
  profile() { return this.hasOne(Profile); }
  roles()   { return this.belongsToMany(Role); }
}

class Post extends Model {
  author()  { return this.belongsTo(User, "user_id"); }
}

Clés personnalisées

// hasMany(related, foreignKey?, localKey?)
this.hasMany(Post, "author_id", "id");

// belongsTo(related, foreignKey?, ownerKey?)
this.belongsTo(User, "author_id", "id");

// belongsToMany(related, pivotTable?, foreignPivotKey?, relatedPivotKey?, parentKey?, relatedKey?)
this.belongsToMany(Role, "role_user", "user_id", "role_id");
Conventions par défaut : clé étrangère = <modèle>_id ; table pivot = noms des deux modèles au singulier, snake_case, triés alphabétiquement et joints par _ (ex. role_user).

Charger une relation

const user = await User.find(1);

// Lazy : exécute la requête maintenant
const posts = await user.posts().getResults();   // Collection<Post>
const profile = await user.profile().getResults(); // Profile | null

// Requête chaînée sur la relation
const recents = await user.posts().where("published", true).orderBy("created_at", "desc").get();

Accès après eager loading

const users = await User.with("posts").get();
for (const u of users) {
  u.relationLoaded("posts");        // true
  const posts = u.getRelation("posts"); // Collection<Post>
}
En TypeScript, user.posts est une méthode (la requête). Pour lire la relation chargée, utilisez user.getRelation("posts") — pas user.posts.

belongsToMany & table pivot

// Attacher / détacher / synchroniser
await user.roles().attach(adminId);
await user.roles().attach([adminId, editorId]);
await user.roles().attach(editorId, { granted: true }); // colonnes pivot supplémentaires
await user.roles().detach(adminId);     // détache un rôle
await user.roles().detach();            // détache tout
await user.roles().sync([viewerId, adminId]); // l'ensemble final = exactement ces ids

// Lire
const roles = await user.roles().getResults();
await user.roles().where("name", "admin").get();

Charger paresseusement après coup

const user = await User.find(1);
await user.load("posts");
user.getRelation("posts");

Filtrer par existence d'une relation

await User.query().has("posts").get();          // a au moins un post
await User.query().doesntHave("posts").get();   // n'a aucun post
await User.query().whereHas("posts", (q) => q.where("published", true)).get();
await User.query().orWhereHas("comments").get();

Fonctionne aussi sur les relations N-N :

await Post.query().whereHas("tags", (q) => q.where("name", "typescript")).get();
